Animation

Il existe 3 balises pour principale pour l'animation en svg, je vais vous les présenter dans ce tutoriel.

La balise<animate/>

Cette balise plusieurs attributs que je vais vous décrire si dessous.

La balise cible de l'animation

    il existe 2 possibilité pour liée ca balise,
  • la 1ere(celle que j'utilise) consite a inclure la balise animate dans la balise de la formes que l'ont souhaite animer.
  • La 2ème possibilité est d'appeler la balise cible par son id avec xlink:href, si vous utiliser cette syntax je vous conseil de mettre la balise animate dans la balise <defs> pour conserver une certaine logique cemantique.

L'attribut ou la propièté cible:

On utilise attributeName=""pour indiqué l'attribut sur lequel on veux influer.Nous devons aussi utiliser attributeType=""pour indiqué de quel language est l'élément sur lequel nous influont, XML ou CSS dans le cas d'animate.

<animate attributeName="x" attributeType="XML">

Les valeurs de début et de fin de l'animation

ils faut maintenant indiqué les valeur de déplacement. Nous allons utiliser from="" pour indiqué la position de départ de l'animation et to=""pour indiquer la position finale de l'animation.Ils est aussi possible d'utilisé l'attribut values mais nous en parlerons plus tard dans le tutoriel

<animate attributeName="x" attributeType="XML" from="20" to="200">

indiquation des valeurs de temps dans l'animation

Comme vous vous en doutez il existe un attribut pour indiqué quand l'animation dois commencer...begin=""cette attribut peux prend des valeurs de temps en hh:mm:ss:ms ou mm:ss:ms ou simplement en seconde

Il faut aussi indiqué une valeurs de duration de l'animation, dur="" il prend aussi des valeurs de temps, avec la même syntaxe.

<animate attributeName="x" attributeType="XML" from="20" to="200" begin="0s" dur="6s">

Stop l'animation a ca position finale

Si l'on test nos animation maintenant on peux constater qu'après avoir effectuer sont animation notre forme reprend ca position initial. Grâce a l'attribut fill="freeze", l'animation sera bloqué dans ca postion finale.

<animate attributeName="x" attributeType="XML" from="20" to="200" begin="0s" dur="6s" fill="freeze">

L'attribut end start

Ces attributs nous permette de synchroniser plus facilement nos animation plus facilement. Un petit exemple: 


<animate id="anim1" attributeName="y" attributeType="XML" from="50" to="100" 
	begin="0s" dur="9s" fill="freeze">
<animate id="anim2" attributeName="x" attributeType="XML" from="15" to="100" 
	begin="#anim1.start+3s" dur="3s" fill="freeze">
<animate id="anim3" attributeName="x" attributeType="XML" from="100" to="15" 
	begin="#anim2.end" dur="3s" fill="freeze">

Les evènement souris

Il existe plusieurs écouteurs d'évenement pour améliorer l'intèractivité:

  • click
  • activate
  • mousedown
  • mouseup
  • mouseover
  • mouseout
  • mousemove

il est aussi possible d'attribué un raccourci clavier(seulement pour lancer une animation) avec begin="accesKey(raccourci)".

L'attribut restart

restart possede 3 valeur, always qui permet de redémarer l'animation n'importe quand, whenNotActivepermet seulement de redamérer l'animation lorsque qu'elle est inactive(terminé) et neverrend le redémarage d'un animation impossible.

Grace a la balise animate que nous avons vu dans la premier chapitre nous pouvons réalisé des translation simple(un seul attribut est modifié: x ou y)...

Raccourci:<set/>

Contrairement a animate elle ne peut pas calculer et dessiner les valeurs comprises entre les valeurs de début et de fin de sont attribut cible, elle n'a donc pas d'attribut dur. Cepedant elle peut s'avérer utile pour l'utilisation de valeurs tel que:

  • Visible
  • hidden
  • collapse

Un exemple:

	
	<path id="triangle" d="M40,80 l50,0 l-25,-70" style="stroke:black">
	<set attributeName="visibility" attributeType="CSS" to="hidden" begin="rectangle.click"/>
	<set attributeName="visibility" attributeType="CSS" to="hidden" begin="rectangle2.click"/>
	<path/>
	<rect id="rectangle" x="20" y="20" height="30" width="30"/>
	<rect id="rectanglebis" x="60"  y="20" height="30" width="30"/>

Les tracé de mouvement

<animateMotion/>

cette balise va nous permettre de réaliser des animation a partir d'un nouvelle attribut path="" nous allons faire bouger nos formes sur des courbes. Dans cette attribut nous allons ecrire un tracé comme on l'a appris dans la chapitre des formes, c'est sur ce tracé que le mouvement s'éffectura(d'ou le nom tracé de mouvement). cette balise ne contiendra donc pas d'attribut: attributeName, attributType, from et to. Cependant comme pour <animate/> elle possède les attribut being et dur:

	
<circle cx="100" cy="100" r="80" style="fill:#ae3" >
	<animateMotion begin="0s" dur="3s" path="M0,0 q40,80 80,160 q200,300 300,50 t200,300 100,-300" fill="freeze"/>
</circle>

<mpath/>

La balise <mpath/>

va nous permetre d'appeler un path avec un appelle d'id, cette balise ne prend qu'un attribut xlink:href="".Ce raccourci est tres utile si l'ont souhaite utiliser plusieurs fois la même path.Un visuel pour bien comprendre 
	
		<svg height="1000px" width="1000px" xmlns="http://www.w3.org/2000/svg"
			xmlns:xlink="http://www.w3.org/1999/xlink">
		<defs>
		<path id="path" d="M0,0 q40,80 80,160 q200,300 300,50 t200,300 100,-300">
		<defs/>

		<ellipse cx="100" cy="100" rx="80" ry="150" style="fill:#ae3">
		<animateMotion begin="1s" dur="3s" fill="freeze">
		<mpath xlink:href="#path">
		</animateMotion>
		</circle>

		<circle cx="200" cy="100" r="80" style="fill:#fae;">
		<animateMotion begin="1.2s" dur="3s" fill="freeze">
		<mpath xlink:href="#path">
		</animateMotion>
		</circle>

		<rect x="300" y="100" height="80" width="80" style="fill:#ea4">
		<animateMotion begin="1.5s" dur="3s" fill="freeze">
		<mpath xlink:href="#path">
		</animateMotion>
		</circle>
